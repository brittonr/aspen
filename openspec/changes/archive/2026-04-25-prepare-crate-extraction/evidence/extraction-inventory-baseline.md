## Baseline observations

- Redb Raft KV first-slice crates are present as workspace packages except the future `aspen-raft-kv-types` and `aspen-raft-kv` package names, which must be introduced or mapped before readiness can be claimed.
- Every row remains `workspace-internal`; license and publication policy are unresolved by design.
- The direct-dependency inventory is only the first pass. Later readiness evidence must add transitive, representative-consumer, feature-unification, and re-export leak checks from `docs/crate-extraction/policy.ncl`.
- Binary-target rows are final consumers. Any reusable behavior they own must move behind library APIs before those behaviors can be considered extractable.
