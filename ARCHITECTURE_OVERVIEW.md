## Execution Backend Abstraction

```rust
pub trait ExecutionBackend: Send + Sync {
    // Identity
    fn backend_type(&self) -> &str;  // "vm", "wasm", "docker", "k8s"

    // Lifecycle
    async fn initialize(&self) -> Result<()>;
    async fn shutdown(&self) -> Result<()>;

    // Execution
    async fn submit_job(&self, job: Job, config: ExecutionConfig) 
        -> Result<ExecutionHandle>;
    async fn get_status(&self, handle: &ExecutionHandle) 
        -> Result<ExecutionStatus>;
    async fn cancel_execution(&self, handle: &ExecutionHandle) 
        -> Result<()>;
    async fn wait_for_completion(&self, handle: &ExecutionHandle, 
        timeout: Option<Duration>) -> Result<ExecutionStatus>;

    // Resource Management
    async fn get_resource_info(&self) -> Result<ResourceInfo>;
    async fn can_handle(&self, job: &Job, config: &ExecutionConfig) 
        -> Result<bool>;

    // Monitoring
    async fn health_check(&self) -> Result<BackendHealth>;
    async fn list_executions(&self) -> Result<Vec<ExecutionHandle>>;
    async fn cleanup_executions(&self, older_than: Duration) 
        -> Result<usize>;

    // Metadata
    async fn get_metadata(&self, handle: &ExecutionHandle) 
        -> Result<ExecutionMetadata>;
}

// This trait allows any execution target!
// âœ… VMs (Firecracker, Cloud Hypervisor)
// âœ… Containers (Docker, OCI)
// âœ… Serverless (Lambda, Cloud Functions)
// âœ… Clusters (Kubernetes, Nomad)
// âœ… Custom (your own orchestrator)
```

## Summary: Why This Architecture is Generic

| Aspect | Implementation | Generality |
|--------|---|---|
| **Job Definition** | Generic JSON payload | Supports any job type |
| **Job Lifecycle** | 5-state machine | Applies to any work |
| **Worker Types** | Pluggable enum | Add new types easily |
| **Execution** | Trait-based backends | Support any executor |
| **Storage** | Generic tables (workflows, workers) | Not CI/CD specific |
| **Events** | Domain events published | Multiple subscribers |
| **Configuration** | Feature-gated | Include only needed parts |
| **Validation** | Composable validators | Custom rules pluggable |
| **Placement** | Strategy trait | Custom matchers |

**Result**: A true distributed job orchestrator that happens to support CI/CD, but is NOT limited to it.
